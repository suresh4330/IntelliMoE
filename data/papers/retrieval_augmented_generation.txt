Title: Retrieval-Augmented Generation for Knowledge-Intensive NLP Tasks
Authors: Lewis, Perez, Piktus, Petroni, Karpukhin, Goyal, Küttler, Lewis, Yih, Rocktäschel, Riedel, Kiela (Facebook AI Research, 2020)
Published: NeurIPS 2020

ABSTRACT
Large pre-trained language models have been shown to store factual knowledge in their parameters, and achieve state-of-the-art results when fine-tuned on downstream NLP tasks. However, their ability to access and precisely manipulate knowledge is still limited. We explore a general-purpose fine-tuning recipe for retrieval-augmented generation (RAG) — models which combine pre-trained parametric and non-parametric memory for language generation.

INTRODUCTION
Knowledge-intensive tasks — tasks that humans could not reasonably be expected to perform without access to an external knowledge source — remain challenging for large language models. The key insight of RAG is that models need not store all facts in parameters; instead, they can retrieve relevant documents at inference time and use them as context to answer questions.

RAG ARCHITECTURE
RAG combines two components:
1. Retriever: A Dense Passage Retriever (DPR) that maps input to a distribution over text passages. Given input x, it retrieves the top-K documents using Maximum Inner Product Search (MIPS) over a pre-built FAISS index of Wikipedia passages.
2. Generator: A seq2seq model (BART) that generates output conditioned on the concatenation of the input and retrieved documents.

  p_RAG(y | x) ≈ Σ_z p_η(z | x) * p_θ(y | x, z)

Where z represents retrieved documents, η parameterizes the retriever, and θ parameterizes the generator.

TWO FORMULATIONS
RAG-Sequence: Uses the same retrieved document to generate the complete sequence. For each retrieved document z, BART generates the full output y, then marginalizes over documents.

RAG-Token: Can use different documents for different tokens. Each output token is generated by marginalizing over all retrieved documents, allowing the model to use different evidence for different parts of the answer.

DENSE PASSAGE RETRIEVAL (DPR)
The retriever encodes both queries and documents using two separate BERT-based encoders:
  E_Q(x): Query encoder (BERT-base)
  E_D(d): Document encoder (BERT-base)
  sim(x, d) = E_Q(x)^T * E_D(d)

The top-K documents are retrieved using MIPS with a FAISS flat index over all Wikipedia passages (21M passages from 100-word blocks).

RESULTS
RAG outperforms parametric seq2seq models and task-specific retrieval-augmented models on:
- Open-domain QA (Natural Questions, TriviaQA, WebQuestions)
- Abstract QA (MS-MARCO)
- Jeopardy question generation
- Fact verification (FEVER)

The model produces more factual, specific, and diverse responses compared to purely parametric baselines.

PRACTICAL IMPLICATIONS FOR RAG SYSTEMS
- Retrieval quality is critical — better retrieved documents → better answers
- Chunk size matters: 100-word chunks work well for factoid QA
- Hybrid retrieval (dense + sparse BM25) often outperforms either alone
- Retrieved context window is limited by generator's maximum input length
- Re-ranking retrieved documents before feeding to the generator improves results

KEY CONTRIBUTIONS
- Introduced RAG as a general framework combining retrieval + generation
- Showed that non-parametric memory can complement parametric LLM knowledge
- Demonstrated strong performance across diverse knowledge-intensive tasks
- Provided a foundation for modern LLM retrieval systems (LangChain, LlamaIndex)
